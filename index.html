The point of this technical document is to educate the reader about some of the basics to using git and github. This is, by no-means a full documentation of git, but merely touches on the very basic functions.

What Is Git

Git is one of the most widely used version control (VCS) systems in the world today. Git is an actively-maintained open source project developed by the creator of the Linux operating system, Linus Torvalds. Git is considered a distributed version control system (DVCS), meaning every developer on a project has a working copy of code, which is also a repository that can contain all changes made to a project by individual team members.

You can think of Git like a more powerful option of saving a document. Unlike a saved document, saving with git records thhe differences in files and keeps a record of each save. You can review changes, examine how a project has grown, and restore previous versions of your code. Paired with GitHub, you can show off your work or collaborate on projects with other devlopers. By pairing Git and Github, you not only keep a backup and change-log of your code on your machine and remotely, but you also keep a day-by-day log of your coding habits. This is useful to add to your portfolio to show potential companies or clients your work. It is very important to get a good understanding of how Git and Github work as Git is considered to be an essential skill for modern developers.

Getting Started With Git

In this documentation, we are going to assume you have Git setup and configured on your machine already. If you haven't yet installed and setup git, you can follow the guide at git-scm.com. which also has plenty of information not listed in this documentation.

The Git Workflow

While using Git, you will quickly realize there is a very standard workflow:

  1. Working Directory
  2. Staging Area
  3. Local Repo (HEAD)
  4. Remote Repo (MAIN)
  5. Repeat

In the following sections, we will go into more detail about the steps of the workflow, along with Git commands to perform the required tasks.
